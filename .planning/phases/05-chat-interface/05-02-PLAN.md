  ```tsx
  <Streamdown plugins={{ code }} isAnimating={isStreaming ?? false} shikiTheme={['github-dark', 'github-dark']}>
    {message.content}
  </Streamdown>
  ```
  - Subtle "Claude" label above in `text-dim text-[10px] uppercase tracking-wider font-medium`
  - When streaming AND isLastAssistant: show a blinking cursor indicator after the content (a small `w-2 h-4 bg-primary animate-pulse inline-block` span)
- **Action buttons** on assistant messages (visible on hover): Copy button (copies message.content to clipboard via `navigator.clipboard.writeText`), Retry button (calls onRetry). Use `Copy` and `RotateCcw` icons from lucide-react. Style: small icon buttons `text-dim hover:text-foreground` appearing on hover of the message row (use group/group-hover pattern).
- Message container: `py-3` padding, no bubbles — flat layout per user decision. Fits within the centered column from ChatPanel.

**Create `src/plugins/chat/MessageList.tsx`** — scrollable message list:
- Props: `{ messages: ChatMessage[], streamingContent: string, isStreaming: boolean, onCopy: (content: string) => void, onRetry: (messageIndex: number) => void, onSuggestedPrompt: (prompt: string) => void }`
- Renders messages in a scrollable container (`overflow-y-auto flex-1`)
- Auto-scroll behavior: scroll to bottom when new messages arrive or streaming content updates, BUT only if user was already near the bottom (within 100px). Track with `useRef` on the scroll container.
- Jump-to-bottom button: if user has scrolled up, show a small floating button at bottom-center "Jump to latest" that scrolls to bottom on click. Style: `bg-surface border border-border text-xs text-muted-foreground hover:text-foreground rounded-full px-3 py-1` positioned absolute at bottom.
- For the streaming message (when `isStreaming && streamingContent`): render a virtual "in-progress" MessageBubble with `role: 'assistant'`, `content: streamingContent`, `isStreaming: true`, `isLastAssistant: true`. This message is NOT in the messages array — it's the live stream.
- Empty state (when `messages.length === 0 && !isStreaming`): centered content with a `MessageCircle` icon (lucide), "Start a conversation" heading, and 2-3 suggested prompt pills (e.g., "Explain this project", "What's the current status?", "Help me debug an issue"). Pills are clickable and call the `onSuggestedPrompt` callback. Style pills as: `border border-border rounded-full px-3 py-1.5 text-xs text-muted-foreground hover:text-foreground hover:border-primary/30 cursor-pointer transition-colors`.

**Create `src/plugins/chat/InputBar.tsx`** — input area with controls:
- Props: `{ onSend: (text: string) => void, onStop: () => void, isStreaming: boolean, disabled?: boolean }`
- Auto-growing `<textarea>` with `rows={1}`, using an `onInput` handler that sets `style.height = 'auto'` then `style.height = Math.min(scrollHeight, 96) + 'px'` (cap at ~4 lines). Set `overflow: 'hidden'` when within cap, `overflow-y: 'auto'` when at max.
- Enter sends (calls `onSend` with trimmed value, resets textarea value and height), Shift+Enter inserts newline (default behavior, no preventDefault)
- When `isStreaming`: replace the Send button with a Stop button (`Square` icon from lucide-react, `bg-destructive/20 text-destructive hover:bg-destructive/30` styling). Stop button calls `onStop`.
- When NOT streaming: show Send button (`Send` icon from lucide-react). Disable when textarea is empty.
- Also handle Escape key during streaming — calls `onStop`.
- Container styling: `border-t border-border bg-surface` bar pinned to bottom of ChatPanel. Inside: flex row with textarea and button. Textarea: `bg-transparent text-foreground placeholder:text-dim focus:outline-none resize-none w-full text-sm`. Placeholder: "Message Claude..."
- Send/Stop button: 32x32 rounded, centered icon.

**Create `src/plugins/chat/ChatPanel.tsx`** — main conversation layout:
- Props: `{ projectId: string, projectPath: string, apiKey: string }`
- Uses `useChatHistory(projectId)` to load persistent messages from Dexie
- Uses `useChatStream(apiKey, projectPath)` for streaming
- **Header**: flex row with "Chat" title (text-sm font-medium), and a "Clear" button (`Trash2` icon from lucide-react, `text-dim hover:text-destructive` styling). Clear button calls `clearProjectHistory(projectId)`.
- **Layout**: flex column, full height. Header at top, MessageList fills middle (`flex-1 min-h-0`), InputBar at bottom. Content area has `max-w-[700px] mx-auto w-full` for centered conversation column per user decision.
- **Send flow** (`handleSend`):
  1. Persist user message to Dexie via `persistMessage({ projectId, role: 'user', content: userMessage, timestamp: Date.now() })`
  2. Build history from current messages: `messages.map(m => ({ role: m.role, content: m.content }))`
  3. Call `sendMessage(history, userMessage)` from useChatStream
  4. When stream completes (status changes to 'done'): persist assistant message to Dexie via `persistMessage({ projectId, role: 'assistant', content: streamingContent, timestamp: Date.now() })` and reset stream. Use a `useEffect` watching `status` to trigger this.
  5. When status is 'aborted': if there's partial streamingContent, persist it as an assistant message too, then reset.
  6. When status is 'error': show an inline error message in the chat (e.g., "Failed to get a response. Check your API key and try again.") — use local state for error display.
- **Retry flow** (`handleRetry(messageIndex)`): Find the user message at messageIndex (or the user message preceding the assistant message), re-send it. This means: call `handleSend` with that user message's content.
- **Copy flow**: `navigator.clipboard.writeText(content)`
- **Loading state**: when `useChatHistory` returns `undefined` (Dexie query in-flight), show a subtle loading skeleton (2-3 animated pulse bars). Do NOT show empty state — distinguish `undefined` (loading) from `[]` (truly empty).
- **Suggested prompt handling**: when a suggested prompt pill is clicked in the empty state, call `handleSend` with that prompt text.
