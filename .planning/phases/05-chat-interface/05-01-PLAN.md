<task type="auto">
  <name>Task 2: Create chat data layer — Dexie DB, Zustand store, streaming hook, and plugin registration</name>
  <files>src/lib/chatDb.ts, src/plugins/chat/chatStore.ts, src/plugins/chat/useChatStream.ts, src/plugins/chat/ChatTab.tsx, src/plugins/chat/index.ts, src/plugins/index.ts</files>
  <action>
1. **Create `src/lib/chatDb.ts`** — Dexie database definition:
   - Define `ChatMessage` interface: `{ id?: number, projectId: string, role: 'user' | 'assistant', content: string, timestamp: number }`
   - Create Dexie instance `chatDb` with `EntityTable<ChatMessage, 'id'>` typing
   - Schema: `messages: '++id, projectId, timestamp'`
   - Export `persistMessage(msg: Omit<ChatMessage, 'id'>)` — adds message then enforces 50-message cap by deleting oldest messages for that projectId if count exceeds MAX_MESSAGES (50)
   - Export `clearProjectHistory(projectId: string)` — deletes all messages for a project
   - Follow the exact Dexie EntityTable pattern from RESEARCH.md code examples

2. **Create `src/plugins/chat/chatStore.ts`** — Zustand store for chat UI state:
   - Interface `ChatState`: `{ isStreaming: boolean, streamingContent: string, error: string | null, setStreaming(v: boolean): void, setStreamingContent(c: string): void, appendStreamingContent(chunk: string): void, resetStream(): void, setError(e: string | null): void }`
   - Create with `create<ChatState>()` — NO persist middleware (streaming state is ephemeral)
   - Export `useChatHistory(projectId: string)` hook that wraps `useLiveQuery` from `dexie-react-hooks`:
     ```typescript
     export function useChatHistory(projectId: string) {
       return useLiveQuery(
         () => chatDb.messages.where('projectId').equals(projectId).sortBy('timestamp'),
         [projectId]
       )
     }
     ```
   - This returns `ChatMessage[] | undefined` (undefined while loading, per Dexie behavior)

3. **Create `src/plugins/chat/useChatStream.ts`** — streaming hook with rAF buffer:
   - Export type `StreamStatus = 'idle' | 'streaming' | 'done' | 'error' | 'aborted'`
   - Export `useChatStream(apiKey: string, projectPath: string)` hook
   - Create Anthropic client ONCE via `useRef` with `{ apiKey, dangerouslyAllowBrowser: true }`. Only create when apiKey is truthy.
   - Implement rAF buffer pattern: `bufferRef` accumulates tokens, `rafRef` tracks the animation frame ID, `flush()` moves buffer to state via `setStreamingContent(prev => prev + bufferRef.current)`
   - `sendMessage(history: Array<{role, content}>, userMessage: string)`:
     - Creates new `AbortController` stored in `abortRef`
     - Resets streaming content to empty string
     - Sets status to 'streaming'
     - Builds messages array: `[...history.slice(-20), { role: 'user', content: userMessage }]` — send last 20 messages for API context (not all 50 from storage)
     - Calls `clientRef.current!.messages.stream()` with: model `'claude-sonnet-4-20250514'`, max_tokens `8096`, system prompt `"You are a helpful assistant. The active project is at ${projectPath}."`, messages array, and `{ signal: abortRef.current.signal }`
     - Uses `stream.on('text', appendToken)` for token buffering
     - Awaits `stream.finalMessage()` then does a final flush and sets status to 'done'
     - Catch: if `err.name === 'AbortError'` set status 'aborted', else set status 'error'
   - `stop()`: calls `abortRef.current?.abort()`
   - Return: `{ streamingContent, status, sendMessage, stop, setStreamingContent }`
   - IMPORTANT: In cleanup (useEffect return), abort any in-flight stream and cancel any pending rAF

4. **Create `src/plugins/chat/ChatTab.tsx`** — stub tab component:

   - On mount, invoke `get_env_var` with `{ name: 'ANTHROPIC_API_KEY' }` to get the API key
