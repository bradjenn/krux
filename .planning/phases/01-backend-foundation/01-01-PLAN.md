**Step 2: Create `backend/src/routes/projects.ts`.**

Create a Hono sub-app exported as `projectsRoute`:

- `GET /` — call `ProjectStore.list()`, return `c.json(projects)`
- `POST /` — parse body `{ path, displayName? }`, validate `path` is present (return 400 if not), call `ProjectStore.add(body.path, body.displayName)`, return 201 with project on success, or 400/409 with error envelope on failure (use `result.code === 'ALREADY_EXISTS' ? 409 : 400` for status code)
- `DELETE /:id` — call `ProjectStore.remove(id)`, return 204 on success, 404 with error envelope on failure
- `GET /:id/phases` — call `ProjectStore.get(id)`, return 404 if null, else call `PlanningParser.parseRoadmap(project.path)` and return JSON
- `GET /:id/state` — call `ProjectStore.get(id)`, return 404 if null, else call `PlanningParser.parseState(project.path)` and return JSON
- `GET /:id/overview` — return `{ project, phases, state }` combined as `ProjectOverview`

All error responses use the locked error envelope: `{ error: { code: string, message: string } }`.

Wrap `c.req.json()` in try/catch to handle malformed JSON (return 400 BAD_REQUEST).
